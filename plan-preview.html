D 数据库设计

Prisma ORM 管理，新建 webchat_v2 数据库，两张核心表：

📋 User 用户表

字段类型说明
idInt主键，自增 PK
emailString邮箱，作为唯一标识 Unique
usernameString显示名称（可重复）
passwordString密码（bcrypt 加密存储）
avatarString头像标识
bioString?个性签名（可选）
createdAtDateTime注册时间

💬 Message 消息表（群聊 + 私聊共用）

字段类型说明
idInt主键，自增 PK
contentText消息内容（文字/Markdown/图片路径）
typeString消息类型：text / image
formatString文本格式：plain / markdown
userIdInt发送者 ID FK → User
targetIdInt?null = 群聊，非null = 私聊目标用户 ID
createdAtDateTime发送时间

💡 群聊和私聊共用一张 Message 表，通过 targetId 区分：null 表示群聊消息，非 null 表示私聊消息。这样历史查询只需要一个接口。

P 私聊方案

实时发送 — Socket.IO 房间机制

两个用户加入专属 room（如 dm:1:2），消息实时推送。在线时即时收到。

离线历史 — 数据库持久化

所有消息实时写入数据库。用户登录后拉取未读消息，离线期间的消息不会丢失。

M 消息渲染与安全

纯文本消息 — plain 格式

使用 DOMPurify 过滤后渲染，防止 XSS 攻击（如 <script> 标签注入）。

Markdown 消息 — markdown 格式

使用 marked 解析为 HTML，再通过 DOMPurify 过滤，支持标题、列表、代码块、加粗等。

⚠️ 安全原则：永远不要直接将用户输入的 HTML 插入 DOM。无论是纯文本还是 Markdown，都必须经过 DOMPurify 过滤后才能渲染。

渲染流程

用户输入 → 判断 format 字段 → plain: 直接过滤
markdown: marked 解析 → DOMPurify 过滤 → 安全渲染到页面

S 实施步骤
1分支与项目骨架
  • git tag v1.0 标记当前版本
  • 创建 v2.0 分支，初始化 client/ 和 server/ 项目
  • 配置 TypeScript、ESLint、Prettier
2后端基础搭建
  • 安装 Express + Socket.IO + Prisma + JWT 依赖
  • 配置 Prisma schema，连接 webchat_v2 数据库
  • 配置 .env 环境变量
3认证系统
  • POST /api/auth/register — 注册（邮箱+用户名+密码+头像+可选个性签名）
  • POST /api/auth/login — 邮箱+密码登录，返回 JWT token
  • JWT 验证中间件保护接口
  • GET /api/user/profile — 获取用户信息（含个性签名）
  • PUT /api/user/profile — 更新个性签名等个人信息
4Socket.IO 实时通信
  • JWT 鉴权：连接时验证 token
  • 群聊：默认 room 广播消息
  • 私聊：专属 room 一对一通信
  • 所有消息实时写入数据库
5聊天历史接口
  • GET /api/chat/history — 查询群聊/私聊历史
  • 支持分页，JWT 鉴权保护
6前端页面开发
  • 登录/注册页 — 表单验证、头像选择
  • 聊天主页 — 侧边栏、消息区、输入框
  • Pinia 状态管理 + Vue Router 路由守卫
7图片持久化
  • multer 文件上传，存到 server/uploads/
  • 图片消息 content 存文件路径
8安全与优化
  • 环境变量外置、CORS 配置
  • 输入校验、Prisma 自动防 SQL 注入
  • 历史消息分页
9收尾
  • 更新 README、.mcp.json
  • 完整流程测试：注册→登录→群聊→私聊→图片→历史
